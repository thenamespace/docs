To interact with the offchain system, you'll need an API key. You can configure your API key either for all domains associated with an address (recommended) or for specific ENS domains.

## Configuring Your API Key

<Tabs>
  <Tab title="Address-Based (Default)">
    Set a default API key that works for all ENS domains associated with your address. This is the recommended approach for most users.

    ```typescript
    // Set default API key for all your domains
    client.setDefaultApiKey('your-api-key');

    // Using an environment variable (recommended)
    client.setDefaultApiKey(process.env.NAMESPACE_API_KEY);
    ```

    The default API key will be used for all domains unless overridden by a domain-specific key.
  </Tab>

  <Tab title="Domain-Specific">
    For cases where you need different API keys for specific domains, you can set them individually:

    ```typescript
    // Set API key for a specific domain
    client.setApiKey("example.eth", "domain-specific-api-key");
    ```

    Domain-specific keys take precedence over the default API key for that particular domain.
  </Tab>
</Tabs>

## API Key Priority

1. Domain-specific API key (if set)
2. Default address-based API key
