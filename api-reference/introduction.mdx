<Note>
  This documentation covers the Offchain Manager API endpoints for managing
  offchain ENS subnames.
</Note>

## Authentication

API endpoints that require authentication use API key authentication via the `x-auth-token` header. You can generate API keys through the [Namespace Dev Portal](https://dev.namespace.ninja/).

```bash
x-auth-token: YOUR_API_KEY
```

Required for:

- Creating subnames
- Deleting subnames

## Key Features

- **Create Subnames**: Register new subnames with text records, addresses, and metadata
- **Retrieve Subnames**: Get detailed information about existing subnames by full name
- **Delete Subnames**: Remove subnames from the system (requires API key)
- **Search Subnames**: Query multiple subnames with flexible filters including parent name, label search, ownership, and metadata
