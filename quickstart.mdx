## What is Namespace & ENS?

Namespace extends [ENS](https://ens.domains/) (Ethereum Name Service) by enabling flexible subname issuance for your community, app, or project. Subnames like `alice.yourdao.eth` provide human-readable identities for users, wallets, or resources.

## Types of Subnames

<Steps>
  <Step title="Offchain Subnames (Recommended)">
    Offchain subnames are gasless, instantly updatable, and ideal for most integrations. Integrate easily with our JS SDK or API.
    <br/>
    <b>Benefits:</b>
    <ul>
      <li>No gas fees</li>
      <li>Instant issuance and updates</li>
      <li>Easy integration with our JS SDK or API</li>
    </ul>
    <Expandable summary="How to issue Offchain Subnames">
      <Steps>
        <Step title="Install the JS SDK">
          <CodeGroup>
            <CodeBlock language="bash" filename="Terminal">
              npm install @namespaceid/offchain-sdk
            </CodeBlock>
          </CodeGroup>
        </Step>
        <Step title="Issue a subname">
          <CodeGroup>
            <CodeBlock language="js" filename="issue-subname.js">
              {`import { issueSubname } from '@namespaceid/offchain-sdk';

const result = await issueSubname({
parent: 'yourdao.eth',
subname: 'alice',
address: '0x123...abc',
apiKey: 'YOUR_API_KEY',
});

console.log(result); // { success: true, subname: 'alice.yourdao.eth' }`}

</CodeBlock>
</CodeGroup>
</Step>
<Step title="API Reference for Other Languages">
See the <a href="/api-reference/offchain/" aria-label="Offchain API Reference">Offchain API Reference</a> for REST API usage and more examples.
</Step>
</Steps>
</Expandable>

  </Step>
  <Step title="Onchain Subnames (L1/L2)">
    Onchain subnames are registered on Ethereum (L1) or supported L2s for full onchain guarantees and ENS compatibility.
    <br/>
    <b>Benefits:</b>
    <ul>
      <li>Maximum decentralization</li>
      <li>Direct ENS compatibility</li>
      <li>Permanent, censorship-resistant records</li>
    </ul>
    <Expandable summary="How to issue Onchain Subnames">
      <Steps>
        <Step title="Install the Mint Manager SDK (JS)">
          <CodeGroup>
            <CodeBlock language="bash" filename="Terminal">
              npm install @namespaceid/mint-manager-sdk
            </CodeBlock>
          </CodeGroup>
        </Step>
        <Step title="Register a subname (JS)">
          <CodeGroup>
            <CodeBlock language="js" filename="register-onchain.js">
              {`import { registerSubname } from '@namespaceid/mint-manager-sdk';

const tx = await registerSubname({
parent: 'yourdao.eth',
subname: 'bob',
owner: '0x456...def',
signer, // ethers.js signer
});

console.log(tx.hash); // Ethereum transaction hash`}

</CodeBlock>
</CodeGroup>
</Step>
<Step title="API Reference for Other Languages">
See the <a href="/api-reference/mint-manager/" aria-label="Mint Manager API Reference">Mint Manager API Reference</a> for REST API usage and more examples.
</Step>
</Steps>
</Expandable>

  </Step>
</Steps>
